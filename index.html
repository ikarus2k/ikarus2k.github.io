이미지보여라

Hexo로 포스트 만들때 이미지 파일은 어디에 위치시키지?

VS Code로 md파일 편집하기

동기

메일로 간단한 설정 문서를 보내고 싶은데 pdf양식으로 보내고 싶고
내용중에는 xml도 있고 shell내용도 있으니 code formatting이 되면 좋겠는데

asciidoc은 복잡하고(maven 으로 가이드 문서 만들때 사용해봤지만…)
Notion으로 하면 편한데. 무료 플랜이라 block을 다써서 PC에서 사용이 안되고 ㅠㅠ
md파일 작성하고 pdf export하는 툴 찾아보니 제대로 안되거나 무료가 아니고 ㅠㅠ

VS Code 좋네

VS Code로 md파일 편집도 되고, Lint도 되고, Preview도 되고, pdf export도 되네. Cool

Win + Shift + s 로 화면 캡처

주로 글쓸때는 화면 캡처해서 올리는 경우가 많고 편하게 하려면
Win10의 Win + Shift + s 버튼으로 화면 캡처하고
클립보드에 저장된걸 바로 문서에 넣으면 편한데
보통은 Win + v 로 붙여넣으면 신세계(카톡이나, word문서 등등)

markdown-image Extenstion

md로 post만들때는 파일을 만들고 참조하도록 해야하니. 완전 귀찮은 상황 but
VS Code에 extension이 있고 markdown-image

Win + Shift +s로 화면 캡처, Ctrl + Alt + v 로 md문서에 붙여넣으면 파일 생성되고 이미지 태그 생성

좋네.. 하지만 github page로 올리면 이미지가 안나오네.

hexo asset-folders

hexo g (generate)해서 hexo d (deply)하면 이미지가 안나오네
이것저것 찾아보다 공식 문서 hexo 문서
asset folder설정대로 하니 성공.

post폴더로 이미지파일 move

hexo new로 post를 만들면 post이름으로 폴더가 생성되고
그 폴더안에 이미지 파일을 넣어두면 잘 찾아지네
(생성된 정적 html의 image link 경로를 날짜/post/ 이런식으로 맞춰줘서 되는듯)

그런데 VS Code에서 preview할때는 이미지가 안보이네. 흠..

toc도 잘나오나

Front-matter

toc는 md문서의 윗 부분 (Front-matter)

Front-matter는 파일 시작 시 작성하는 YAML 또는 JSON 구역입니다.
게시물에 대한 환경 설정을 이 곳에서 합니다.
Front-matter가 끝나는 부분은 YAML의 경우 세 개의 대시(-) 로, JSON의 경우 세 개의 세미콜론(;)을 넣어서 구분합니다.
